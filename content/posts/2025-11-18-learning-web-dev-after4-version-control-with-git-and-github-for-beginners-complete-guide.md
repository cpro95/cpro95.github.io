# 웹 개발 배우기 번외 4편 - 협업과 백업의 필수품, Git과 GitHub 완벽 가이드

이 글은 프로그래밍 경험이 없는 분들을 대상으로 자바스크립트 웹 앱 제작법을 알려드리는 '웹 개발 배우기' 시리즈의 일부인데요.


이번 챕터에서는 버전 관리 시스템인 '깃(Git)'과 유용한 동반자 웹사이트인 '깃허브(GitHub)'를 사용하는 방법을 배울 건데요.

이 두 가지는 팀으로 프로그래밍할 때 중요한 도구이지만, 혼자 작업하는 프로그래머에게도 큰 도움이 됩니다.


## 깃(Git)이란 무엇일까요?
깃(Git)은 '버전 관리 시스템'인데요.

저장소(디렉토리라고 생각하세요)에 있는 파일들의 여러 버전을 그들의 역사에 걸쳐 관리합니다.

깃(Git)은 `.html`이나 `.css` 같은 텍스트 파일과 가장 잘 작동하지만, 바이너리 파일도 처리할 수 있습니다.


## 깃(Git) 설치하기
네이티브 패키지 매니저를 사용해 `git` 셸 명령어를 설치하세요.

예를 들면 다음과 같습니다.

*   **윙겟(WinGet)**: `Git.Git`

*   **홈브루(Homebrew)**: `git`


설치 후에는 다음 명령어를 실행해주세요.

```bash
git config --global init.defaultBranch main
```
이것은 일반적인 설정이며, 나중에 사용할 `git init` 사용 시 표시되는 팁 메시지를 비활성화합니다.


## 로컬에서 깃(Git) 사용하기
이 섹션에서는 파일이 있는 디렉토리를 깃(Git) 저장소로 바꾸고, 그런 저장소를 사용하는 방법을 탐색하는데요.

깃허브(GitHub)를 탐색할 때 이 파일들을 다시 사용할 것이니, 이 섹션이 끝난 후 삭제하지 마세요.


### 파일이 있는 로컬 디렉토리 만들기
다음과 같은 파일들을 만들겠습니다.

*   `git-demo/README.md`

*   `git-demo/LICENSE`

*   `git-demo/public/index.html`


`README.md` 파일은 저장소에 무엇이 들어있는지 설명하는데요.

`LICENSE` 파일은 다른 사람들이 여러분의 콘텐츠로 무엇을 할 수 있고 무엇을 할 수 없는지 명확히 해줍니다.

'오픈 소스 라이선스 선택하기' 웹사이트가 여러분의 프로젝트에 좋은 라이선스를 찾는 데 도움을 줄 수 있습니다.


### 디렉토리를 깃(Git) 저장소로 바꾸기
이제 우리 디렉토리를 깃(Git) 저장소로 바꿀 차례인데요.

```bash
% cd git-demo
% git init
Initialized empty Git repository in /tmp/git-demo/.git/
```
셸을 통해 깃(Git)을 사용할 때, 우리는 항상 `git` 명령어를 호출하는데요.

`git init`과 같은 많은 하위 명령어를 가지고 있습니다.

이 명령어는 `git-demo/.git/` 디렉토리를 추가하는데, 이곳은 깃(Git)이 이 저장소를 관리하는 데 필요한 데이터를 저장하는 곳입니다.


`git status` 하위 명령어는 우리 저장소의 상태에 대한 정보를 제공합니다.


### 저장소에 커밋하기
깃(Git) 저장소에 변경 사항을 '커밋(저장)'하는 것은 두 단계로 이루어지는데요.

1.  **변경 사항을 스테이징합니다**: `git add`로 추적하지 않는 파일이나 변경된 추적 파일을 추가합니다.

2.  **스테이징된 변경 사항을 커밋합니다**: 변경 사항을 설명하는 '커밋 메시지'와 함께 저장소에 커밋합니다.


`README.md`를 우리 저장소에 커밋해 봅시다.

먼저 스테이징하고요.

```bash
% git add README.md
```
그런 다음 스테이징된 변경 사항을 커밋합니다 (`-m` 옵션은 커밋 메시지를 지정합니다).

```bash
% git commit -m "First commit"
```
`git log`는 우리 저장소에 대한 커밋 기록을 보여줍니다.


### 추적하지 않을 파일
대부분의 저장소에는 우리가 절대 추적하고 싶지 않은 파일들이 포함되어 있는데요.

저장소의 최상위 레벨에 `.gitignore` 파일을 만들어 깃(Git)에게 그 파일들이 존재하지 않는 척하라고 말할 수 있습니다.


## 원격 깃(Git) 저장소
깃(Git)은 단순한 버전 관리 시스템이 아니라, '분산' 버전 관리 시스템인데요.

각 로컬 저장소는 0개 이상의 '원격 저장소'를 가질 수 있습니다.


원격 저장소의 목적은 무엇일까요?

우리는 로컬 데이터를 원격 데이터와 동기화 상태로 유지하고 싶습니다.

일반적인 시나리오는 다음과 같습니다.

*   단일 사용자가 로컬 저장소를 서버의 원격 저장소와 동기화합니다 (클라우드 백업).

*   팀원들이 서버의 공유 저장소에서 작업합니다.


다음 용어들이 중요한데요.

*   **풀링(Pulling)**: 원격 저장소에서 변경 사항을 가져와 로컬 저장소에 병합하는 것을 의미합니다.

*   **푸싱(Pushing)**: 변경 사항을 원격 저장소로 보내고 그곳에 병합하는 것을 의미합니다.

*   **클로닝(Cloning)**: 원격 저장소를 복사하여 로컬 저장소로 만드는 것을 의미합니다.


## 비주얼 스튜디오 코드를 통해 깃(Git) 사용하기
비주얼 스튜디오 코드(Visual Studio Code)에는 깃(Git)을 위한 훌륭한 그래픽 사용자 인터페이스가 있는데요.

저장소를 생성하거나 클로닝할 때는 셸을 사용하고, 그 외에는 가능한 한 비주얼 스튜디오 코드(Visual Studio Code)를 사용하는 것을 추천합니다.


## 깃허브(GitHub) 사용하기
깃허브(GitHub)는 깃(Git) 저장소를 호스팅하고 그 저장소 주변의 다양한 서비스를 제공하는 웹사이트인데요.


### 깃허브(GitHub) 계정 만들고 로그인하기
1.  github.com으로 가서 계정을 만듭니다.

2.  패키지 매니저를 사용해 깃허브(GitHub) CLI(셸 명령어 `gh`)를 설치합니다.

3.  `gh auth login`을 사용해 계정 전체를 깃허브(GitHub)에 로그인합니다.


깃허브(GitHub)에 로그인하면 무엇을 얻을 수 있을까요?

매번 로그인할 필요 없이 깃허브(GitHub) 저장소에 변경 사항을 푸시할 수 있다는 것을 의미합니다.


### 유용한 `gh` 명령어 더 알아보기
우리가 이미 본 유용한 `gh` 명령어 외에, 제가 사용하는 유일한 다른 두 가지는 다음과 같은데요.

*   로그인 상태 정보: `gh auth status`

*   저장소 클로닝: `gh repo clone <user-name>/<repo-name>`


### 깃허브(GitHub) 저장소 만들기
1.  github.com으로 가서 새 저장소를 만듭니다.

2.  `gh repo clone`을 사용해 저장소를 여러분의 드라이브에 클로닝합니다.

3.  `git-demo/`의 파일들을 새 디렉토리로 옮깁니다.

4.  비주얼 스튜디오 코드(Visual Studio Code)를 사용해 모든 파일을 로컬 저장소에 커밋하고, 이 새 커밋을 원격 저장소로 푸시합니다.

5.  github.com에서 여러분의 원격 저장소를 확인해보세요.

    커밋한 파일들이 보일 겁니다.


## 왜 깃(Git)을 사용해야 할까요?
단일 사용자는 무한한 실행 취소가 가능한 자동 백업을 얻게 되는데요.

이는 실수로 데이터를 잃어버릴 걱정을 절대 할 필요가 없다는 것을 의미합니다.

언제든지 과거의 상태로 돌아갈 수 있거든요.


팀은 앞서 언급된 이점과 더불어, 깃(Git)이 공유된 아티팩트 집합에서 작업하는 것을 훨씬 더 쉽게 만든다는 이점을 얻습니다.


## 더 읽을거리
우리는 깃(Git)이 어떻게 작동하는지에 대해 겨우 표면만 긁었지만, 여러분이 배운 기본 사항은 비교적 멀리까지 갈 수 있게 해줄 건데요.

깃(Git)에 대한 더 많은 정보는 git-scm.com의 "문서"를 참조하세요.
