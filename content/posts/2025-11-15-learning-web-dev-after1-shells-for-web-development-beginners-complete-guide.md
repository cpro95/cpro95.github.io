이 글은 프로그래밍 경험이 없는 분들을 대상으로 자바스크립트 웹 앱 제작법을 알려드리는 '웹 개발 배우기' 시리즈의 일부인데요.

'셸'은 운영체제와 텍스트 기반으로 상호작용하는 방법을 제공하는데요.

이번 챕터에서는 셸이 어떻게 작동하는지, 그리고 웹 개발에 왜 필요한지 탐색해 볼 겁니다.


## 파일 시스템 경로
우리가 필요로 할 용어 중 익숙하지 않을 수 있는 것들이 있는데요.

*   **디렉토리**: 폴더의 다른 말입니다.

    파일 시스템에서 파일을 포함하는 항목입니다.

*   **홈 디렉토리**: 사용자의 고유한 파일들을 포함합니다.

    컴퓨터의 각 사용자는 이러한 디렉토리를 가지고 있습니다.


파일 시스템 경로가 어떻게 작동하는지 빠르게 복습해 보겠습니다.

경로는 파일과 디렉토리의 위치를 지정하는데요.

'절대 경로'와 '상대 경로' 두 종류가 있습니다.


'절대 경로'는 파일의 위치를 직접 지정하는데요.

예를 들어, 다음 절대 경로는 사용자 `robin`의 홈 디렉토리를 참조합니다.

*   **윈도우**: `C:\Users\robin\`

*   **유닉스 (맥OS, 리눅스)**: `/Users/robin/` 또는 `/home/robin/`


유닉스 운영체제는 경로 세그먼트를 슬래시(`/`)로 구분하는 반면, 윈도우는 역슬래시(`\`)를 사용합니다.

윈도우도 구분자로 슬래시를 허용하기 때문에, 지금부터는 종종 슬래시를 사용할 겁니다.


대략적으로 '상대 경로'는 `C:\`(윈도우)나 `/`(유닉스)로 시작하지 않는 경로인데요.

현재 디렉토리의 절대 경로에 상대적으로 해석됩니다.

현재 디렉토리의 경로와 상대 경로를 결합하면 절대 경로를 얻게 되죠.


상대 경로의 예시는 다음과 같습니다.

*   현재 디렉토리의 파일: `main.js`

*   현재 디렉토리의 부모: `..`

*   부모 디렉토리의 파일: `../notes.txt`

*   현재 디렉토리: `.`


## 셸은 커맨드 라인 인터페이스를 가진 파일 시스템 탐색기입니다
그래픽 사용자 인터페이스(GUI)는 운영체제와 상호작용하는 가장 인기 있는 방법이 되었는데요.

프로그램을 시작하려면 그래픽 파일 시스템 탐색기에서 아이콘을 더블클릭하고, 파일을 이동하려면 아이콘을 한 창에서 다른 창으로 드래그합니다.


'커맨드 라인 사용자 인터페이스(CLI)'는 다르고 더 오래된 방식인데요.

텍스트 명령어를 입력하고, 거의 항상 텍스트인 결과를 얻습니다.

웹 개발에 필요한 많은 도구들이 커맨드 라인 사용자 인터페이스를 가지고 있거든요.

왜 그럴까요?

자주 실행되는 작업을 자동화하기 쉽기 때문입니다.


'셸'은 대략적으로 GUI 파일 시스템 탐색기와 관련된 기능을 가진 CLI 앱인데요.

우리는 이른바 '터미널 앱'(줄여서 터미널)의 텍스트 창 안에서 셸을 실행할 겁니다.


## 셸에서 현재 디렉토리 변경하고 파일 목록 보기
셸을 시작하면, 처음에는 프롬프트가 표시되는데요.

`%`나 `>`와 같은 표시로, 입력을 기다리고 있음을 알려줍니다.

우리가 사용할 수 있는 몇 가지 명령어는 다음과 같습니다.


| 기능 | 윈도우 | 유닉스 |
|---|---|---|
| 현재 디렉토리 경로 표시 | `pwd` | `pwd` |
| 현재 디렉토리 파일 목록 | `dir` | `ls` |
| 다른 현재 디렉토리로 이동 | `cd` | `cd` |
| 텍스트 파일 내용 표시 | `type` | `cat` |

예를 들어, 한 디렉토리 위로 올라가는 것은 다음과 같습니다.

`cd ..`

이 경우 `cd`는 명령어이고 `..`는 그 인자입니다.


`index.html` 파일의 내용을 표시하는 예시는 다음과 같습니다.

*   윈도우: `type index.html`

*   유닉스: `cat index.html`


셸은 훨씬 더 많은 것을 할 수 있는데요.

이 시리즈에서는 위에서 배운 것보다 훨씬 더 많은 것이 필요하지는 않습니다.

새로운 셸 기능이 필요할 때마다, 미리 설명해 드릴 겁니다.

## 셸 사용 팁
*   **히스토리**: 위쪽 화살표 키를 누르면 이전에 입력했던 명령어들을 다시 볼 수 있습니다.

*   **폴더 경로 확인하기**: 때로는 운영체제의 그래픽 파일 시스템 탐색기에서 본 폴더의 경로를 셸에서 사용하고 싶을 때가 있는데요.

    그 폴더를 터미널 창으로 끌어다 놓으면 해당 경로를 삽입할 수 있습니다.
